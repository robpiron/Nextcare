[Guida] Come distribuire NextCare su un Web Server in produzione?

Questa guida descrive i passi necessari per ospitare la piattaforma NextCare su un server web reale (Intranet o Cloud), connetterla ad un server MySQL di produzione e configurare lo scambio LIS con percorsi di rete UNC.

1. Architettura di Produzione

La piattaforma è composta da:

  • Frontend (Client Statico): File HTML, CSS e JS (index.html, style.css, app.js) caricati dal browser dell'utente.
  • Backend (API Server): Script Python (server.py) in ascolto sulla porta 8000.
  • Database Relazionale: Server MySQL (porta 3306) che memorizza tutti i dati dell'applicazione.
  • Cartelle di Scambio LIS (Middleware): Percorsi di rete UNC (es. \\server\condivisa\import).

2. Configurazione Iniziale (Setup Wizard)

Al primo accesso, viene caricata la procedura d'installazione guidata automatica. Inserisci i parametri MySQL e i percorsi di rete UNC condivisi. Cliccando su "Installa & Inizializza", lo script popolerà il database da schema.sql ed eseguirà il seeding.

3. Guida Passaggio per Passaggio

  1. Copiare i File sul Server: Crea una cartella sul server di destinazione (es. C:\inetpub\wwwroot\NextCare o /var/www/nextcare) e copia tutti i file di NextCare.
  2. Configurare le Cartelle di Scambio LIS (UNC): Crea la cartella principale NextCare_LIS_Exchange con all'interno export e import, e fornisci i permessi di lettura/scrittura all'utente del servizio ed al middleware di laboratorio.
  3. Configurare Python in Background (Windows con NSSM): 
    nssm.exe install NextCareService
    Configura il path verso python.exe e passagli server.py come argomento.
  4. Configurare Python in Background (Linux con Systemd): Crea /etc/systemd/system/nextcare.service: 
    [Service]
WorkingDirectory=/var/www/nextcare
ExecStart=/usr/bin/python3 server.py
Restart=always
  5. Configurare il Reverse Proxy (Nginx): Reindirizza le chiamate statiche alla directory principale, e crea un proxy pass per le API: 
    location /api/ {
    proxy_pass http://127.0.0.1:8000;
}

Questa guida illustra visivamente i passaggi chiave per il caricamento, la configurazione iniziale ed il funzionamento dell'architettura di rete di NextCare.

1. Schema Architetturale di Produzione e Flusso LIS

Il diagramma seguente mostra come i client (browser di medici e operatori) comunicano con il Web Server locale (tramite Reverse Proxy Nginx/IIS), il quale si collega a MySQL e dialoga con il middleware LIS leggendo e scrivendo nelle cartelle condivise di rete (percorsi UNC).

2. Configurazione Iniziale (Setup Wizard)

Al primo avvio, NextCare avvia automaticamente il setup guidato per inserire i parametri di connessione e connettere il database. I dati del seed vengono importati al termine dell'installazione.

[LIS] Come creare e gestire i Fogli di Lavoro LIS?
I fogli di lavoro raggruppano specifici esami per velocizzare il lavoro quotidiano del tecnico di laboratorio:
1. Configurazione: Vai in Configurazione Prestazioni > Fogli di lavoro LIS. Clicca su Aggiungi Foglio, assegna un nome (es. "Ematologia") e seleziona gli esami da includere (puoi filtrare la lista digitando nel campo di ricerca). Salva il foglio.
2. Data Entry: Vai nel modulo Laboratorio (LIS) > Fogli di lavoro LIS. Seleziona il foglio desiderato ed il periodo temporale. Il sistema mostrerà la griglia dei campioni contenenti quegli esami, permettendoti di digitare i risultati inline e stamparli cliccando su Stampa Foglio.
[LIS] Come esportare le richieste ed importare i risultati LIS (XML)?
NextCare comunica in modalità bidirezionale con i middleware di laboratorio tramite file standard XML:
- Esportazione Richieste: Clicca su Esporta Richieste XML LIS nel modulo LIS. Il sistema genererà un file XML con le anagrafiche pazienti e i codici esame accettati per inviarli agli analizzatori.
- Importazione Risultati: Clicca su Importa Risultati XML LIS. Il server leggerà i file XML prodotti dagli analizzatori posizionati nella cartella import, abbinando i risultati ai campioni tramite barcode ed aggiornandoli in tempo reale.
[LIS] Come si inseriscono, modificano e validano i risultati del laboratorio?
Puoi immettere o modificare i risultati delle analisi in due modi:
- Da Elenco Campioni: Individua il campione, clicca su Inserisci Risultati, inserisci i valori e clicca su Salva per inviarli alla validazione medica.
- Da Fogli di Lavoro LIS: Seleziona il foglio e inserisci i valori direttamente nelle caselline della griglia. Lo stato del test passerà automaticamente a completed (o flagged se fuori range di riferimento) e il campione diventerà Da Validare.
- Validazione: Il medico del laboratorio rivede i valori e clicca su Valida & Firma Referto per renderlo ufficiale.
[LIS] Come ristampare le etichette con Codice a Barre (Barcode) dei campioni?
Le etichette termiche con codice a barre e dati del paziente vengono proposte automaticamente alla chiusura dell'accettazione. Se hai necessità di ristamparle:
1. Vai in Laboratorio (LIS) > Elenco Campioni/Provette.
2. Cerca il campione tramite codice a barre, nome paziente o codice fiscale usando i filtri in alto.
3. Fai clic sul pulsante **Azioni** o sull'icona della stampante/etichetta (Ristampa Etichetta) per aprire la finestra di stampa termica del barcode.
[LIS] Come modificare o annullare una sessione di accettazione LIS?
Le accettazioni di laboratorio (o sessioni di prelievo) possono essere corrette o eliminate:
- Modifica Sessione: Vai in Laboratorio (LIS) > Accettazioni LIS (Sessioni). Trova la sessione d'interesse e clicca su Modifica per cambiare il medico prescrittore, i dettagli del paziente o gli esami richiesti.
- Annullamento: Se la provetta non è ancora stata processata, puoi cliccare su Elimina (o Annulla Accettazione) per rimuovere la richiesta dal flusso di lavoro degli analizzatori e stornare la relativa fattura.
[Guida V1.0] FSE 2.0 & Privacy (Logiche di Oscuramento)

In conformità con le direttive del Garante della Privacy e le specifiche tecniche del Fascicolo Sanitario Elettronico 2.0 (FSE 2.0), NextCare integra le logiche di oscuramento delle prestazioni sanitarie sia in fase di accettazione amministrativa che in fase di refertazione clinica.

1. Configurazione in Accettazione (CUP)

Durante la prenotazione o il check-in di un esame, l'operatore può indicare la volontà dell'assistito selezionando lo stato di oscuramento desiderato:

  • P99 (Oscuramento Totale): La prestazione non sarà visibile a nessun operatore sanitario al di fuori della struttura erogatrice.
  • P97 (Oscuramento a Familiari/Tutori): La prestazione sarà oscurata per gli esercenti la responsabilità genitoriale o tutori (utile per minori o soggetti sotto tutela).
  • P98 (Oscuramento Provvisorio): La prestazione è provvisoriamente oscurata all'assistito stesso (ad esempio in attesa di consulto specialistico guidato).
  • Nessun Oscuramento (Null): La prestazione sarà visibile sul FSE secondo le regole di consenso generali dell'assistito.

2. Conferma in Refertazione (LIS / RIS)

Il medico firmatario, nella maschera di redazione e firma del referto, visualizza la scelta effettuata in accettazione. Il medico ha la facoltà di confermare o modificare lo stato di oscuramento (ad esempio per motivi clinici urgenti o su richiesta esplicita del paziente al momento della visita) prima di apporre la firma digitale.

3. Generazione e Trasmissione del Documento

Al momento della firma, il backend di NextCare esegue automaticamente le seguenti operazioni:

  1. Generazione CDA XML: Viene prodotto il file HL7 CDA R2 XML contenente i dati clinici codificati (LOINC, ecc.). Se è presente un codice di oscuramento, l'elemento confidentialityCode dell'header CDA viene impostato sul codice di restrizione V (Very Restricted), altrimenti viene impostato su N (Normal).
  2. Iniezione PDF: Utilizzando la libreria pypdf, il file cda.xml viene iniettato come file allegato all'interno del referto PDF firmato.
  3. Generazione Token JWT & Invio REST: Viene generato il Bearer Token firmato asimmetricamente con la chiave privata e inviato all'endpoint SOGEI, includendo il codice di oscuramento (es. P99) nel parametro attiCliniciRegoleAccesso del JSON di metadati della chiamata.
[Guida V1.0] Accettazione SSN & Caricamento Ricette (NRE)

NextCare supporta pienamente l'accettazione di prestazioni in regime di convenzione con il Servizio Sanitario Nazionale (SSN) tramite impegnativa dematerializzata.

Processo di Accettazione Ricetta

  1. Inserimento Numero Ricetta Elettronica (NRE): Digita il codice NRE e il Codice Fiscale del paziente nel pannello di accettazione SSN.
  2. Rilevamento Esenzioni: Il sistema esegue un lookup automatico sulla tabella ministeriale delle esenzioni (caricata da esenzioni.xlsx). Se il paziente ha diritto a un'esenzione (es. per reddito, patologia o età), il ticket a suo carico viene azzerato o ridotto, tranne per la quota fissa ricetta ove prevista dalla normativa regionale.
  3. Lookup ASL Automatico: Inserendo la città di residenza del paziente, NextCare calcola l'ASL di competenza basandosi sul mapping dei comuni italiani (caricato da asl.xlsx), inserendo automaticamente il codice ASL corretto (es. "ASL Milano" -> 101).
  4. Regola di Split Ricette (Max 8 prestazioni): Per legge, una singola ricetta dematerializzata può contenere al massimo 8 prestazioni dello stesso ramo specialistico. Se l'accettazione include più di 8 esami (tipico nel LIS), NextCare suddivide automaticamente le prestazioni in ricette separate (es. 12 esami verranno suddivisi in una ricetta da 8 e una da 4 esami), calcolando i ticket distinti.
  5. Calcolo Ticket e Tetti Massimi: Il sistema calcola il ticket per ogni prestazione applicando le tariffe del nomenclatore nazionale (caricato da nomenclatore.xlsx) fino a un massimo di 36,15 € a ricetta, più la quota fissa regionale (es. 10 € per ricetta).
[Guida V1.0] Rendicontazione ASL (Export Flussi Mensili C/M & Report DCR)

La rendicontazione delle attività erogate in regime SSN avviene a cadenza mensile tramite la generazione di flussi telematici standard inviati alla ASL regionale.

Tipologie di Flusso e Report

  • Flusso C (Prestazioni Ambulatoriali): Esporta il file di testo posizionale contenente l'elenco di tutte le prestazioni di specialistica ambulatoriale e diagnostica strumentale erogate ai pazienti SSN nel periodo selezionato.
  • Flusso M (Medicina Specialistica): Esporta il flusso relativo alle prestazioni di laboratorio analisi cliniche.
  • Report DCR (Documento Contabile Riepilogativo): Genera il documento DCR contenente il riepilogo finanziario del mese (totale importo rimborsabile ASL, totale ticket riscossi, quote fisse) necessario per l'emissione della fattura di rimborso alla ASL.

Procedura di Invio Mensile

  1. Accedi al menu Rendicontazione ASL & Flussi.
  2. Seleziona il Mese e l'Anno di riferimento.
  3. Clicca su Valida Dati per verificare che non ci siano anomalie (es. CF mancanti, esenzioni invalide, diagnosi assenti).
  4. Clicca su Genera Flussi C/M per scaricare i file strutturati da caricare sul portale regionale (Edotto, SaniARP, ecc.).
  5. Scarica il Report DCR PDF per l'ufficio contabilità.
[Guida V1.0] Spesa Sanitaria STS 730 (Invio Telematico)

Il Sistema Tessera Sanitaria (STS) impone l'invio delle spese sanitarie sostenute dai cittadini per consentire la predisposizione della dichiarazione dei redditi precompilata (730).

Regole di Invio e Filtri

  • Filtro Pagamento: Vengono inviate al Sistema TS esclusivamente le fatture in stato "Pagata" (saldate). Le fatture in sospeso non vengono trasmesse finché non viene registrato l'incasso.
  • Opposizione del Paziente: Se il paziente ha espresso opposizione all'invio (tramite apposito checkbox in accettazione), la fattura viene trasmessa contrassegnando l'apposito flag di opposizione, tutelando la privacy del cittadino.
  • Modalità di Invio: NextCare invia i dati in modalità sincrona/asincrona tramite Web Service SOAP/MTOM. In locale, lo scambio produce i file XML pronti all'interno della cartella di esportazione C:\NextCare_STS_Exchange\export.
[Guida V1.0] Configurazione ASL/STS (Credenziali & Certificati)

Prima di poter effettuare le trasmissioni telematiche al FSE 2.0 e al Sistema TS, è necessario configurare le credenziali della struttura.

Dati Richiesti

  • Codice Regione / ASL / Struttura: Assegnati dalla regione di appartenenza.
  • Username / Password / PIN: Credenziali personali del medico o della struttura rilasciate dal Sistema Tessera Sanitaria.
  • FSE 2.0 Gateway e Certificati: L'endpoint del gateway (sandbox/produzione), il Vendor ID del software, il Repository OID e i percorsi assoluti ai file di certificato client (.pem) e chiave privata (.key) necessari per l'autenticazione mutua (mTLS).
[Guida V1.0] Gestione Compensi Medici (Configurazione Regole & Payout)

NextCare include un motore di calcolo compensi flessibile per medici e specialisti, strutturato in base a contratti specifici.

1. Configurazione delle Regole

Per ogni medico è possibile definire più regole di calcolo nella scheda Contratti & Compensi:

  • Branca / Servizio: La regola può essere generale (tutta la branca, es. Cardiologia) o specifica per singola prestazione (es. Elettrocardiogramma).
  • Tipo Calcolo: Percentuale (es. 40% del prezzo della prestazione) o Valore Fisso (es. 25,00 € a prestazione).

2. Eventi di Calcolo (Trigger)

Il compenso si attiva automaticamente in corrispondenza di specifici eventi del workflow:

  • Erogato: Calcolato quando la prestazione viene marcata come eseguita dal terapista o TSRM.
  • Refertato: Calcolato alla firma del referto medico.
  • Fatturato: Calcolato all'emissione della fattura al paziente.
  • Incassato: Calcolato esclusivamente quando la fattura viene saldata (stato Pagata).

Il sistema seleziona sempre la regola a massima specificità (la regola sul singolo servizio prevale su quella di branca).

[Guida V1.0] Integrazione PACS/DICOM & Modality Worklist (MWL)

Il modulo RIS/PACS consente la comunicazione bidirezionale con i macchinari diagnostici (RM, TC, RX, Ecografi) e i server PACS.

1. Modality Worklist (MWL)

NextCare include un server MWL integrato attivo sulla porta 11104 con AE Title NEXTCARE_RIS. I macchinari possono inviare una query C-FIND (es. inserendo la data odierna) per scaricare l'elenco degli appuntamenti prenotati per quella modalità. L'operatore sul macchinario seleziona il paziente direttamente dalla lista, eliminando errori di digitazione di nome, cognome o CF.

2. C-GET e Visualizzatore Web

Il medico radiologo può eseguire una ricerca (C-FIND) sul PACS centralizzato inserendo filtri (data, paziente, tipo esame) e avviare il retrieve delle immagini tramite C-GET. NextCare visualizza le immagini DICOM direttamente nel browser tramite canvas interattivo, consentendo zoom, regolazione di luminosità/contrasto (window width/center) e misurazioni lineari.

[Guida V1.0] Portali Web Pazienti & Aziende Partner

I portali web dedicati offrono un punto di accesso riservato all'esterno della rete clinica per la consultazione e il download dei documenti.

1. Portale Paziente

Ogni paziente registrato in anagrafica riceve automaticamente le credenziali di accesso (Codice Fiscale come Username, e una password generata dal sistema o impostata manualmente). Accedendo al portale, il paziente può:

  • Scaricare i propri referti LIS/RIS firmati digitalmente.
  • Scaricare le fatture in formato PDF.
  • Cancellare autonomamente gli appuntamenti prenotati entro 24/48 ore dall'evento.

2. Portale Aziende (Medicina del Lavoro)

Le aziende convenzionate o clienti della medicina del lavoro accedono inserendo Partita IVA e password. Nel portale possono scaricare i certificati di idoneità lavorativa cumulativi rilasciati dal medico competente per i propri dipendenti ed accedere alle fatture a carico dell'azienda.

[Guida V1.0] Gestione Magazzino, Lotti & Catena del Freddo

Il modulo di logistica garantisce il tracciamento dei consumabili clinici, dei reagenti e delle temperature di stoccaggio dei farmaci.

1. Carico/Scarico Merci (DDT)

Ogni movimento di magazzino viene registrato compilando il modulo Documento di Trasporto (DDT) specificando il fornitore, il lotto del reagente/farmaco, la data di scadenza e la quantità caricata o scaricata.

2. Avvisi di Scadenza e Sotto-scorta

NextCare monitora le scorte inviando notifiche in tempo reale sulla dashboard in caso di lotti in scadenza nei successivi 30 giorni o quantità inferiori alla scorta minima impostata per ciascun articolo.

3. Log Frigoriferi (Catena del Freddo)

I frigoriferi per la conservazione di reagenti e campioni biologici sono monitorati registrando periodicamente la temperatura (intervallo ottimale 2-8 °C). In caso di scostamento, il sistema logga l'anomalia e allerta i responsabili del laboratorio.

[Guida V1.0] Cicli di Riabilitazione & Scala VAS

Il modulo Riabilitazione gestisce le terapie fisiche e i cicli di rieducazione motoria/funzionale dei pazienti.

1. Configurazione Ciclo & Diari Clinici

Il ciclo viene attivato impostando il numero totale di sedute (es. 10 sedute di tecarterapia). Per ogni seduta eseguita, il fisioterapista inserisce a diario:

  • La descrizione degli esercizi o del trattamento praticato.
  • Note sull'evoluzione clinica del paziente.
  • La valutazione del dolore del paziente tramite la Scala VAS (Visual Analogue Scale) da 0 a 10.

2. Regole di Fatturazione

Il ciclo può essere fatturato secondo due modalità stabilite in fase di accettazione:

  • Fatturazione Anticipata (Start): L'intero ciclo viene fatturato all'inizio del trattamento.
  • Fatturazione Posticipata (End): Il ciclo viene fatturato e saldato all'ultima seduta eseguita.
Come posso modificare liberamente una tariffa durante l'accettazione?
Durante il check-in di un appuntamento (CUP/RIS/Visite) o la registrazione di un campione (LIS), i prezzi delle prestazioni compaiono all'interno di campi di testo editabili. È sufficiente digitare la tariffa desiderata. Il sistema mostrerà un popup di avviso per confermare la variazione e ricalcolerà automaticamente sconti ed eventuali franchigie assicurative. La nuova tariffa verrà memorizzata nella fattura specifica.
Come funziona la sincronizzazione tra LocalStorage e database MySQL?
L'applicazione NextCare funziona in doppia modalità. Se il database MySQL è configurato, tutti i dati vengono salvati localmente nel browser (LocalStorage) e sincronizzati automaticamente con il server MySQL in tempo reale. Se la connessione al database fallisce o si lavora in modalità offline (porta 8001), l'applicazione continuerà a funzionare esclusivamente in LocalStorage senza perdere dati.
Come importare i risultati delle analisi dal middleware LIS?
Accedi alla scheda Prelievi & LIS e clicca su Importa Risultati XML LIS. Il sistema analizzerà i file XML posizionati nella cartella di importazione configurata (default: C:\NextCare_LIS_Exchange\import), importerà i risultati nel database associandoli al campione tramite il codice a barre (barcode) e sposterà i file elaborati nella sottocartella processed.
Come posso configurare o modificare i tipi di provette biologiche?
Vai nel menu Configurazione Prestazioni e clicca sulla sottoscheda Gestione Provette LIS. Lì troverai una griglia che elenca tutte le provette configurate con il loro nome, colore identificativo e il suffisso numerico utilizzato per generare i codici a barre (barcode) univoci dei campioni. Puoi aggiungere nuove tipologie o eliminare/modificare quelle esistenti.
Dove trovo la configurazione dell'integrazione DICOM / PACS?
La configurazione PACS si trova nella scheda Database SQL sotto la sezione Configurazione Nodi DICOM e Server PACS. Qui puoi impostare l'AE Title del Viewer locale, la porta di ascolto ed i parametri del server PACS remoto (IP, Porta, AE Title) per le query ed il retrieve delle immagini diagnostiche.
Come posso prenotare una prestazione o un esame per un paziente?
Vai nel menu Accettazione & CUP e fai clic su Nuova Prenotazione. Seleziona il paziente (o creane uno nuovo premendo "+"), compila la data e l'ora, scegli la prestazione e il medico esecutore. Fai clic su "Salva Prenotazione". L'appuntamento apparirà in stato "Prenotata" sia in agenda che nella cronologia del CUP.
Come utilizzare il calendario e l'agenda delle visite mediche?
Accedi alla sezione Calendario & Agende. Lì puoi selezionare un giorno specifico con il calendario mensile e visualizzare le colonne orarie di ciascun medico attivo. Gli slot colorati rappresentano gli appuntamenti. Cliccando su uno slot puoi visualizzare i dettagli della prenotazione, modificarla o procedere all'accettazione diretta. Puoi filtrare l'agenda per branca specialistica o per singolo medico.
Come creare o modificare le agende e gli orari di lavoro dei medici?
Vai nella sezione Medici & Agende e clicca sulla scheda Pianificazione Agende. Qui puoi creare una nuova agenda settimanale per un medico specificando i giorni della settimana, l'ora di inizio/fine lavoro, la durata media di ciascuna visita (slot orari) e le prestazioni autorizzate per quel medico. Puoi anche configurare i turni di presenza nella scheda Turni Staff.
Come creare o aggiungere nuovi esami o prestazioni mediche?
Vai su Configurazione Prestazioni, rimani sulla scheda Elenco Prestazioni e fai clic su Nuova Prestazione. Compila la scheda inserendo il Codice CUR, il nome della prestazione, la tipologia (Visita, LIS o RIS), la branca medica e la tariffa standard di base. Se si tratta di un esame LIS, puoi anche associare il tipo provetta specifico e definire i singoli parametri analitici con i rispettivi range di riferimento e unità di misura.
Come gestire le pratiche assicurative (Fatturazione Unica vs Fatturazione Split)?
Quando registri un'accettazione per un paziente con una convenzione assicurativa aperta (collegata a una pratica/claim), puoi selezionare la pratica nel menu a tendina. Il sistema calcolerà l'eventuale franchigia a carico del paziente e la quota a carico dell'assicurazione basandosi sulle regole della convenzione.

- Se la modalità di fatturazione concordata con l'ente è Fattura Unica (Single), verrà emessa una sola fattura co-intestata con l'indicazione delle quote.
- Se la modalità è Fattura Distinta (Split), il sistema genererà automaticamente due fatture separate: una intestata al paziente per la sua quota/franchigia, e una intestata all'ente assicurativo per l'importo coperto. Entrambe le fatture sono tracciate singolarmente nel pannello contabile e nei report di fatturazione.

    Documentazione e FAQ Integrazione XML LIS

    Questa guida descrive le specifiche di integrazione bidirezionale XML tra NextCare e i middleware / strumenti di laboratorio (es. SynlabNET).

    1. Directory di Scambio Windows

    Lo scambio dei file avviene localmente sul PC/Server Windows tramite le seguenti cartelle predefinite:

    • Esportazione richieste: C:\NextCare_LIS_Exchange\export\
    • Importazione risultati (da scansionare): C:\NextCare_LIS_Exchange\import\
    • File elaborati con successo: C:\NextCare_LIS_Exchange\import\processed\
    • File con errori di sintassi o parsing: C:\NextCare_LIS_Exchange\import\errors\

    2. Esportazione Richieste LIS (NextCare -> Middleware)

    All'esportazione, viene creato un file XML con nomenclatura [data_giorno]-NextCare.xml.

    <?xml version="1.0" encoding="utf-8"?> <ListaRichieste> <CodiceLaboratorio>NEXTCARE</CodiceLaboratorio> <Richiesta> <Cognome>Rossi</Cognome> <Nome>Mario</Nome> <IdRichiestaLis>000000020001</IdRichiestaLis> <!-- Barcode della richiesta --> <Sesso>M</Sesso> <!-- M o F --> <DataNascita>1980-01-01</DataNascita> <Data>2026-06-11</Data> <!-- Data accettazione --> <CodiceFiscale>RSSMRA80A01H501U</CodiceFiscale> <!-- Opzionale --> <Analisi Codice="90.12.3" IdCampioneLis="000000020001" Materiale="SIERO" /> </Richiesta> </ListaRichieste>

    3. Importazione Risultati (Middleware -> NextCare)

    I sistemi terzi devono depositare i file XML con i risultati nella cartella di importazione. NextCare supporta sia file a singola richiesta sia a richieste multiple.

    <?xml version="1.0" encoding="utf-8"?> <ListaRichieste> <Richiesta> <IdRichiestaLis>000000020001</IdRichiestaLis> <!-- Barcode corrispondente --> <Analisi Codice="90.12.3"> <!-- Codice dell'analisi --> <Risultato Codice="COL"> <!-- Codice parametro --> <Descrizione>Colesterolo Totale (LIS)</Descrizione> <Valore>210</Valore> <UDM>mg/dL</UDM> <LimiteInferiore>120</LimiteInferiore> <!-- Opzionale --> <LimiteSuperiore>200</LimiteSuperiore> <!-- Opzionale --> <FuoriNorma>true</FuoriNorma> <!-- true o false --> </Risultato> </Analisi> </Richiesta> </ListaRichieste>

    4. Logica di Associazione e Validazione

    • Barcode (IdRichiestaLis): Deve coincidere esattamente con il codice a barre stampato/generato da NextCare durante l'accettazione del prelievo.
    • Associazione dei parametri: NextCare effettua una ricerca basata sul codice prestazione ed il nome dell'analita. In caso di codice parametro (es: COL), questo viene confrontato case-insensitive per garantire una corrispondenza corretta.
    • Stato Post-Importazione: Una volta importato il file, lo stato del campione passa automaticamente a "Da Validare" nel pannello LIS. I valori fuori norma vengono contrassegnati in rosso come allerta per l'operatore e il medico.
